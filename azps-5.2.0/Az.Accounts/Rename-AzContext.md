# Rename-AzContext

## STRESZCZENIe
Zmienianie nazwy kontekstu platformy Azure.  Domyślnie nazwy kontekstowe są nazywane przez konto użytkownika i abonament.

## POLECENIA

### RenameByInputObject (domyślny)
```
Rename-AzContext -InputObject <PSAzureContext> [-Force] [-PassThru] [-Scope <ContextModificationScope>]
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [-TargetName] <String> [<CommonParameters>]
```

### RenameByName
```
Rename-AzContext [-Force] [-PassThru] [-Scope <ContextModificationScope>]
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [-SourceName] <String> [-TargetName] <String>
 [<CommonParameters>]
```

## Opis
Zmienianie nazwy kontekstu platformy Azure.  Domyślnie nazwy kontekstowe są nazywane przez konto użytkownika i abonament.

## Przykłady

### Przykład 1: Zmienianie nazwy kontekstu przy użyciu nazwanych parametrów
```powershell
PS C:\> Rename-AzContext -SourceName "[user1@contoso.org; 12345-6789-2345-3567890]" -TargetName "Work"
```

Zmień nazwę kontekstu dla user1@contoso.org subskrypcji "12345-6789-2345-3567890" na "praca".  Po wykonaniu tego polecenia będziesz mieć możliwość kierowania kontekstu przy użyciu funkcji "Select-AzContext Work".  Zwróć uwagę, że możesz przełączać się między wartościami "SourceName" przy użyciu funkcji kończenia karty.

### Przykład 2: Zmienianie nazwy kontekstu przy użyciu parametrów pozycyjnych
```powershell
PS C:\> Rename-AzContext "My context" "Work"
```

Zmień nazwę kontekstu o nazwie "mój kontekst" na "praca".  Po tym poleceniu będziesz mieć możliwość kierowania kontekstu przy użyciu Select-AzContext

# Stop-AzDeployment

## STRESZCZENIe
Anulowanie uruchomionego wdrożenia

## POLECENIA

### StopByDeploymentName (domyślny)
```
Stop-AzDeployment [-Name] <String> [-PassThru] [-Pre] [-DefaultProfile <IAzureContextContainer>] [-WhatIf]
 [-Confirm] [<CommonParameters>]
```

### StopByDeploymentId
```
Stop-AzDeployment -Id <String> [-PassThru] [-Pre] [-DefaultProfile <IAzureContextContainer>] [-WhatIf]
 [-Confirm] [<CommonParameters>]
```

### StopByInputObject
```
Stop-AzDeployment -InputObject <PSDeployment> [-PassThru] [-Pre] [-DefaultProfile <IAzureContextContainer>]
 [-WhatIf] [-Confirm] [<CommonParameters>]
```

## Opis
Polecenie cmdlet **stop-AzDeployment** anuluje wdrożenie w zakresie subskrypcji, który rozpoczął się, ale nie został ukończony.
Aby zatrzymać wdrożenie, wdrożenie musi mieć niekompletny stan inicjowania obsługi, taki jak obsługa administracyjna, a nie stan wykonania, na przykład zainicjowana obsługa administracyjna lub niepowodzenie.

Aby utworzyć wdrożenie w zakresie abonamentu, użyj polecenia cmdlet New-AzDeployment.

To polecenie cmdlet powoduje zatrzymanie tylko jednego uruchomionego wdrożenia. Użyj parametru *name* , aby zatrzymać określone wdrożenie.

## Przykłady

### Przykład 1
```
PS C:\>Stop-AzDeployment -Name "deployment01"
```

To polecenie powoduje anulowanie uruchomionego wdrożenia "deployment01" w bieżącym zakresie subskrypcji.

### Przykład 2
```
PS C:\>Get-AzDeployment -Name "deployment01" | Stop-AzDeployment
```

To polecenie pobiera wdrożenie "deployment01" w bieżącym zakresie subskrypcji i anuluje je.

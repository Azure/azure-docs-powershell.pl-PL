# Stop-AzManagementGroupDeployment

## STRESZCZENIe
Anulowanie uruchomionego wdrożenia w grupie zarządzania

## POLECENIA

### StopByDeploymentName (domyślny)
```
Stop-AzManagementGroupDeployment [-ManagementGroupId] <String> [-Name] <String> [-PassThru] [-Pre]
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

### StopByDeploymentId
```
Stop-AzManagementGroupDeployment -Id <String> [-PassThru] [-Pre] [-DefaultProfile <IAzureContextContainer>]
 [-WhatIf] [-Confirm] [<CommonParameters>]
```

### StopByInputObject
```
Stop-AzManagementGroupDeployment -InputObject <PSDeployment> [-PassThru] [-Pre]
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

## Opis
Polecenie cmdlet **stop-AzManagementGroupDeployment** powoduje anulowanie wdrożenia, które zostało uruchomione, ale nie zostało ukończone w grupie zarządzania.
Aby zatrzymać wdrożenie, wdrożenie musi mieć niekompletny stan inicjowania obsługi, taki jak obsługa administracyjna, a nie stan wykonania, na przykład zainicjowana obsługa administracyjna lub niepowodzenie.

Aby utworzyć wdrożenie w grupie zarządzania, użyj polecenia cmdlet New-AzManagementGroupDeployment.

## Przykłady

### Przykład 1
```
PS C:\>Stop-AzManagementGroupDeployment -ManagementGroupId "myMG" -Name "deployment01"
```

To polecenie powoduje anulowanie uruchomionego wdrożenia "deployment01" w grupie zarządzania "myMG".

### Przykład 2
```
PS C:\>Get-AzManagementGroupDeployment -ManagementGroupId "myMG" -Name "deployment01" | Stop-AzManagementGroupDeployment
```

To polecenie pobiera wdrożenie "deployment01" w grupie zarządzania "myMG" i anuluje je.

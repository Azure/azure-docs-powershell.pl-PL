# Enable-AzSqlServerAdvancedDataSecurity

## STRESZCZENIe
Włączanie zaawansowanych zabezpieczeń danych na serwerze.

## POLECENIA

```
Enable-AzSqlServerAdvancedDataSecurity [-DoNotConfigureVulnerabilityAssessment] [-AsJob]
 [-DeploymentName <String>] [-InputObject <AzureSqlServerModel>] -ServerName <String>
 [-ResourceGroupName] <String> [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm]
 [<CommonParameters>]
```

## Opis
Polecenie cmdlet **enable-AzSqlServerAdvancedDataSecurity** włącza zaawansowane zabezpieczenia danych na serwerze. Zaawansowane zabezpieczenia danych to ujednolicony pakiet zabezpieczeń, który obejmuje klasyfikację danych, ocenę luk i zaawansowaną ochronę przed zagrożeniami dla serwera. (Nowe konto magazynu zostanie automatycznie utworzone w celu zapisania ocen dotyczących luk. Jeśli w tym celu utworzono wcześniej konto magazynu, zamiast tego zostanie użyte to polecenie.

## Przykłady

### Przykład 1: Włączanie zaawansowanych zabezpieczeń danych serwera
```powershell
PS C:\>  Enable-AzSqlServerAdvancedDataSecurity `
            -ResourceGroupName "ResourceGroup01" `
            -ServerName "Server01" 

ResourceGroupName            : ResourceGroup01
ServerName                   : Server01
IsEnabled                    : True
```

### Przykład 2: Włączanie zaawansowanych zabezpieczeń danych serwera z zasobu serwera
```powershell
PS C:\>  Get-AzSqlServer `
           -ResourceGroupName "ResourceGroup01" `
           -ServerName "Server01" `
           | Enable-AzSqlServerAdvancedDataSecurity

ResourceGroupName            : ResourceGroup01
ServerName                   : Server01
IsEnabled                    : True
```

### Przykład 3

Włączanie zaawansowanych zabezpieczeń danych na serwerze. (autogenerowana)

```powershell
<!-- Aladdin Generated Example --> 
Enable-AzSqlServerAdvancedDataSecurity -DoNotConfigureVulnerabilityAssessment -ResourceGroupName MyResourceGroup -ServerName s1
```

# Invoke-AzSqlDatabaseFailover

## STRESZCZENIe
Praca awaryjna bazy danych.

## POLECENIA

```
Invoke-AzSqlDatabaseFailover [-DatabaseName] <String> [-ReadableSecondary] [-AsJob] [-PassThru] [-Force]
 [-ServerName] <String> [-ResourceGroupName] <String> [-DefaultProfile <IAzureContextContainer>] [-WhatIf]
 [-Confirm] [<CommonParameters>]
```

## Opis
Polecenie cmdlet Invoke-AzSqlDatabaseFailover tryb failover bazy danych Azure SQL Database. Jeśli baza danych znajduje się w puli elastycznej, to polecenie przejdzie w tryb failover dla danej bazy danych bez wpływu na inne bazy danych w tej samej puli elastycznej.

## Przykłady

### Przykład 1
```powershell
PS C:\> Invoke-AzSqlDatabaseFailover -ResourceGroupName "ResourceGroup01" -ServerName "Server01" -DatabaseName "Database01"
```

To polecenie przejdzie w tryb failover replika podstawowa bazy danych o nazwie "Database01" na serwerze o nazwie "Server01".

### Przykład 2
```powershell
PS C:\> Invoke-AzSqlDatabaseFailover -ResourceGroupName "ResourceGroup01" -ServerName "Server01" -DatabaseName "Database01" -ReadableSecondary
```

To polecenie przejdzie w tryb failover do czytelnej repliki pomocniczej bazy danych o nazwie "Database01" na serwerze o nazwie "Server01".

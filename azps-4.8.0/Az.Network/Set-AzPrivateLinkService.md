# Set-AzPrivateLinkService

## STRESZCZENIe
Umożliwia zaktualizowanie usługi linku prywatnego.

## POLECENIA

```
Set-AzPrivateLinkService -PrivateLinkService <PSPrivateLinkService> [-AsJob]
 [-DefaultProfile <IAzureContextContainer>] [<CommonParameters>]
```

## Opis
Polecenie cmdlet **Set-AzPrivateLinkService** umożliwia zaktualizowanie usługi linku prywatnego.

## Przykłady

### 1: Tworzenie usługi linku prywatnego i aktualizowanie jej
```
$vnet = Get-AzVirtualNetwork -ResourceName "myvnet" -ResourceGroupName "myresourcegroup"
$IPConfig = New-AzPrivateLinkServiceIpConfig -Name "IP-Config" -Subnet $vnet.subnets[1] -PrivateIpAddress "10.0.0.5"
$publicip = Get-AzPublicIpAddress -ResourceGroupName "myresourcegroup"
$frontend = New-AzLoadBalancerFrontendIpConfig -Name "FrontendIpConfig01" -PublicIpAddress $publicip
$lb = New-AzLoadBalancer -Name "MyLoadBalancer" -ResourceGroupName "myresourcegroup" -Location "West US" -FrontendIpConfiguration $frontend  
$privateLinkService = New-AzPrivateLinkService -ServiceName "mypls" -ResourceGroupName myresourcegroup -Location "West US" -LoadBalancerFrontendIpConfiguration $frontend -IpConfiguration $IPConfig

$newIPConfig = New-AzPrivateLinkServiceIpConfig -Name "New-IP-Config" -Subnet $vnet.subnets[0] 
$privateLinkService.IpConfigurations[0] = $newIPConfig
$privateLinkService | Set-AzPrivateLinkService
```

W tym przykładzie jest tworzona usługa linku prywatnego o nazwie mypls. Następnie zastępuje on swoje elementy Ipconfiguration z obiektu ipConfiguratiuon w pamięci. Polecenie cmdlet Set-AzPrivateLinkService jest następnie używane do napisania zmodyfikowanego stanu usługi link prywatny po stronie usługi.

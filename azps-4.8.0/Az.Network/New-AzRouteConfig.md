# New-AzRouteConfig

## STRESZCZENIe
Tworzy trasę dla tabeli tras.

## POLECENIA

```
New-AzRouteConfig [-Name <String>] [-AddressPrefix <String>] [-NextHopType <String>]
 [-NextHopIpAddress <String>] [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm]
 [<CommonParameters>]
```

## Opis
Polecenie cmdlet **New-AzRouteConfig** tworzy trasę dla tabeli routingu platformy Azure.

## Przykłady

### Przykład 1. Tworzenie trasy
```powershell
PS C:\>$Route = New-AzRouteConfig -Name "Route07" -AddressPrefix 10.1.0.0/16 -NextHopType "VnetLocal"
PS C:\> $Route
Name              : Route07
Id                : 
Etag              : 
ProvisioningState : 
AddressPrefix     : 10.1.0.0/16
NextHopType       : VnetLocal
NextHopIpAddress  :
```

Pierwsze polecenie tworzy trasę o nazwie Route07, a następnie zapisuje ją w zmiennej $Route.
Ta trasa przekazuje pakiety do lokalnej sieci wirtualnej.
Drugie polecenie wyświetla właściwości trasy.

### Przykład 2

Tworzy trasę dla tabeli tras. (autogenerowana)

<!-- Aladdin Generated Example -->
```powershell
New-AzRouteConfig -AddressPrefix 10.1.0.0/16 -Name 'Route07' -NextHopIpAddress '12.0.0.5' -NextHopType 'VnetLocal'
```

### -NextHopType
Określa, jak ta trasa przekazuje pakiety dalej.
Dopuszczalne wartości tego parametru to:
- Internetowe.
Domyślna Brama internetowa oferowana przez usługę Azure. 
- Znaleziono.
Jeśli określisz tę wartość, trasa nie przesyła dalej pakietów. 
- VirtualAppliance.
Urządzenie wirtualne dodane do sieci wirtualnej platformy Azure. 
- VirtualNetworkGateway.
Brama wirtualnej sieci prywatnej w usłudze Azure Server-to-Server. 
- VnetLocal.
Lokalna sieć wirtualna.
Jeśli masz dwie podsieci, 10.1.0.0/16 i 10.2.0.0/16 w tej samej sieci wirtualnej, wybierz wartość VnetLocal dla każdej podsieci, aby przekazać ją do innej podsieci.

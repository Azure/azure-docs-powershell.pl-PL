# Update-AzWebAppTrafficRouting

## STRESZCZENIe
Aktualizowanie reguły routingu do gniazda.

## POLECENIA

```
Update-AzWebAppTrafficRouting -ResourceGroupName <String> -WebAppName <String> -RoutingRule <Hashtable>
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

## Opis
Polecenie cmdlet **Update-AzWebAppTrafficRouting** aktualizuje konfigurację reguły routingu dla miejsca w aplikacji Azure Web App.

## Przykłady

### Przykład 1 Aktualizowanie reguły routingu w celu przeniesienia 15% ruchu produkcyjnego do gniazda STG
```powershell
PS C:\>Update-AzWebAppTrafficRouting -ResourceGroupName "Default-Web-WestUS" -WebAppName "ContosoSite" 
- RoutingRule @{AtionHostName='XXXX.azurewebsites.net';ReroutePercentage=15;Name='Stg'}
```

To polecenie powoduje zaktualizowanie reguły routingu w celu przeniesienia 15% ruchu produkcyjnego do gniazda STG.

### Przykład 2 aktualizowanie reguły routingu w celu przetransferowania ruchu produkcyjnego do gniazd STG z zakresu od 50% do 90% w sposób przyrostowy.
```powershell
PS C:\>Update-AzWebAppTrafficRouting -ResourceGroupName "Default-Web-WestUS" -WebAppName "ContosoSite" 
-RoutingRule @{ActionHostName='XXXX.azurewebsites.net';ReroutePercentage=50;ChangeIntervalInMinutes=1;
MinReroutePercentage=50;MaxReroutePercentage=90;Name='Stg';ChangeStep=10}
```

To polecenie aktualizuje regułę routingu w celu przetransferowania ruchu produkcyjnego do zakresów STG od 50% do 90% w sposób przyrostowy.

# New-AzureRmDdosProtectionPlan

## STRESZCZENIe
Tworzy plan ochrony DDoS.

[!INCLUDE [migrate-to-az-banner](../../includes/migrate-to-az-banner.md)]

## POLECENIA

```
New-AzureRmDdosProtectionPlan -ResourceGroupName <String> -Name <String> -Location <String> [-Tag <Hashtable>]
 [-AsJob] [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

## Opis
Polecenie cmdlet New-AzureRmDdosProtectionPlan tworzy plan ochrony DDoS.

## Przykłady

### Przykład 1: Tworzenie i kojarzenie planu ochrony DDoS z nową siecią wirtualną
```
D:\> $ddosProtectionPlan = New-AzureRmDdosProtectionPlan -ResourceGroupName ResourceGroupName -Name DdosProtectionPlanName -Location "West US"
D:\> $subnet = New-AzureRmVirtualNetworkSubnetConfig -Name SubnetName -AddressPrefix 10.0.1.0/24
D:\> $vnet = New-AzureRmvirtualNetwork -Name VnetName -ResourceGroupName ResourceGroupName -Location "West US" -AddressPrefix 10.0.0.0/16 -DnsServer 8.8.8.8 -Subnet $subnet -EnableDdoSProtection -DdosProtectionPlanId $ddosProtectionPlan.Id
```

Najpierw tworzymy nowy plan ochrony DDoS za pomocą polecenia **New-AzureRmDdosProtectionPlan** .
Następnie tworzymy nową sieć wirtualną z **nowymi — AzureRmvirtualNetwork** i określimy identyfikator nowo utworzonego planu w parametrze **DdosProtectionPlanId**. W tym przypadku ze względu na to, że kojarzy sieć wirtualną z planem, możemy także określić parametr **EnableDdoSProtection**.

### Przykład 2: Tworzenie i kojarzenie planu ochrony DDoS z istniejącą siecią wirtualną
```
D:\> $ddosProtectionPlan = New-AzureRmDdosProtectionPlan -ResourceGroupName ResourceGroupName -Name DdosProtectionPlanName -Location "West US"
D:\> $vnet = Get-AzureRmVirtualNetwork -Name VnetName -ResourceGroupName ResourceGroupName
D:\> $vnet.DdosProtectionPlan = New-Object Microsoft.Azure.Commands.Network.Models.PSResourceId
D:\> $vnet.DdosProtectionPlan.Id = $ddosProtectionPlan.Id
D:\> $vnet.EnableDdosProtection = $true
```

Najpierw tworzymy nowy plan ochrony DDoS za pomocą polecenia **New-AzureRmDdosProtectionPlan** .
Następnie uzyskujemy najnowszą wersję sieci wirtualnej, którą chcemy skojarzyć z planem. Aktualizacja właściwości **DdosProtectionPlan** z obiektem **PSResourceId** zawierającym odwołanie do identyfikatora nowo utworzonego planu. W tym przypadku, jeśli skojarzię sieć wirtualną z planem ochrony DDoS, możemy również ustawić flagę **EnableDdosProtection** na true (prawda).
Na koniec nowy stan utrzymuje się, gdy zmienna lokalna jest ustawiona na **Set-AzureRmVirtualNetwork**.

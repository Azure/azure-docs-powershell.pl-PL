# New-AzVMWareAuthorization

## STRESZCZENIe
Tworzenie lub aktualizowanie autoryzacji obwodu ExpressRoute w chmurze prywatnej

## POLECENIA

```
New-AzVMWareAuthorization -Name <String> -PrivateCloudName <String> -ResourceGroupName <String>
 [-SubscriptionId <String>] [-DefaultProfile <PSObject>] [-AsJob] [-NoWait] [-Confirm] [-WhatIf]
 [<CommonParameters>]
```

## Opis
Tworzenie lub aktualizowanie autoryzacji obwodu ExpressRoute w chmurze prywatnej

## Przykłady

### Przykład 1: Create Autorization
```powershell
PS C:\> New-AzVMWareAuthorization -Name azps-test-auth -PrivateCloudName azps-test-cloud -ResourceGroupName azps-test-group



Name           Type
----           ----
azps-test-auth Microsoft.AVS/privateClouds/authorizations
```

To polecenie cmdlet umożliwia utworzenie autoryzacji `azps-test-auth` w ramach prywatnej chmury `azps-test-cloud`

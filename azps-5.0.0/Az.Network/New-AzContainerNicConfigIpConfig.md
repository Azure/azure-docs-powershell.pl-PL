# New-AzContainerNicConfigIpConfig

## STRESZCZENIe
Umożliwia utworzenie obiektu konfiguracji IP konfiguracji interfejsu karty sieciowej kontenera.

## POLECENIA

```
New-AzContainerNicConfigIpConfig -Name <String> -Subnet <PSSubnet> [-SubnetId <String>]
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

## Opis
Polecenie cmdlet **New-AzContainerNicConfigIpConfig** tworzy nową konfigurację adresu IP konfiguracji interfejsu sieciowego kontenera. 

## Przykłady

### Przykład 1
```powershell
$subnet = New-AzVirtualNetworkSubnetConfig -Name subnet -AddressPrefix 10.0.1.0/24

$vnet = New-AzVirtualNetwork -Name vnet -ResourceGroupName rg1 -Location "West US" -AddressPrefix 10.0.0.0/16 -Subnet $subnet

$containerNicConfigIpConfig = New-AzContainerNicConfigIpConfig -Name ipconfigprofile1 -Subnet $vnet.Subnets[0]

$containerNicConfig = New-AzContainerNicConfig -Name cnic -IpConfiguration containerNicConfigIpConfig

$networkProfile = New-AzNetworkProfile -Name np1 -Location "West US" -ResourceGroupName rg1 -ContainerNetworkInterfaceConfiguration $containerNicConfig
```

Pierwsze dwa polecenia tworzą i inicjują sieć VNET oraz podsieć. W trzecim poleceniu jest tworzony profil konfiguracji IP kontenera, który odwołuje się do utworzonej podsieci. Czwarte polecenie tworzy konfigurację interfejsu sieciowego kontenera dostarczającego profil konfiguracji IP utworzony w poprzednim poleceniu. Wreszcie, piąte polecenie tworzy profil sieciowy zainicjowany z konfiguracją interfejsu sieciowego kontenera przechowywaną w $containerNicConfig.

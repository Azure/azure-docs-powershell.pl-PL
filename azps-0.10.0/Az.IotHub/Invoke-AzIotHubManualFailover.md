# Invoke-AzIotHubManualFailover

## STRESZCZENIe
Wywoływanie procesu przełączania awaryjnego dla Centrum IoT w regionie odzyskiwania po awarii geograficznej.

## POLECENIA

### ResourceSet (domyślny)
```
Invoke-AzIotHubManualFailover [-ResourceGroupName] <String> [-Name] <String> [-AsJob] [-PassThru]
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

### InputObjectSet
```
Invoke-AzIotHubManualFailover [-InputObject] <PSIotHub> [-AsJob] [-PassThru]
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

### ResourceIdSet
```
Invoke-AzIotHubManualFailover [-ResourceId] <String> [-AsJob] [-PassThru]
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

## Opis
Wyzwoli przejęcie awaryjne swojego centrum IoT w lokalizacji pomocniczej. Ta czynność spowoduje powstanie czasu i utraty danych telemetrycznych w rozwiązaniu. Jest to operacja długotrwała, która może trwać kilka minut. Podczas korzystania z tej funkcji należy zachować ostrożność.

## Przykłady

### Przykład 1
```powershell
PS C:\> Invoke-AzIotHubManualFailover -ResourceGroupName "myresourcegroup" -Name "myiothub"
```

Inicjowanie procesu przełączania awaryjnego Centrum IoT myiothub.

### Przykład 2
```powershell
PS C:\> Invoke-AzIotHubManualFailover -ResourceGroupName "myresourcegroup" -Name "myiothub" -AsJob
```

Inicjowanie procesu przełączania awaryjnego Centrum IoT "myiothub" w tle.

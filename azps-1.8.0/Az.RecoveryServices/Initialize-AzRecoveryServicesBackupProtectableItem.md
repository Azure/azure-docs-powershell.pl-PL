# Initialize-AzRecoveryServicesBackupProtectableItem

## STRESZCZENIe
To polecenie wyzwala odnajdowanie wszelkich niechronionych elementów danego typu obciążenia w danym kontenerze. Jeśli aplikacja bazy danych nie jest chroniona automatycznie, użyj tego polecenia, aby odkryć nowe DBs, ilekroć zostaną dodane i kontynuować ochronę.

## POLECENIA

```
Initialize-AzRecoveryServicesBackupProtectableItem [-Container] <ContainerBase> [-WorkloadType] <WorkloadType>
 [-VaultId <String>] [-DefaultProfile <IAzureContextContainer>] [<CommonParameters>]
```

## Opis
polecenie cmdlet bada konkretne obciążenia w kontenerze. Spowoduje to wyzwolenie operacji, która tworzy elementy, które mogą być chronione.

## Przykłady

### Przykład 1
```
PS C:\> Initialize-AzRecoveryServicesProtectableItem -Container $Container –WorkloadType “MSSQL”
```

Polecenie cmdlet wykonuje operację wykrywania nowych elementów chronionych.

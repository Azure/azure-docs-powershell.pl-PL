# Stop-AzManagementGroupDeployment

## SYNOPSIS
Anulowanie uruchomionego wdrożenia w grupie zarządzania

## SKŁADNIA

### StopByDeploymentName (Domyślna)
```
Stop-AzManagementGroupDeployment [-ManagementGroupId] <String> [-Name] <String> [-PassThru] [-Pre]
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

### StopByDeploymentId
```
Stop-AzManagementGroupDeployment -Id <String> [-PassThru] [-Pre] [-DefaultProfile <IAzureContextContainer>]
 [-WhatIf] [-Confirm] [<CommonParameters>]
```

### StopByInputObject
```
Stop-AzManagementGroupDeployment -InputObject <PSDeployment> [-PassThru] [-Pre]
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

## OPIS
Polecenie cmdlet **Stop-AzManagementGroupDeployment** anuluje wdrożenie rozpoczęte, ale nie ukończone w grupie zarządzania.
Aby zatrzymać wdrożenie, wdrożenie musi mieć niepełny stan inicjowania obsługi, taki jak inicjowanie obsługi administracyjnej, a nie stan ukończony, taki jak Inicjowanie obsługi administracyjnej lub Zakończone niepowodzeniem.

Aby utworzyć wdrożenie w grupie zarządzania, użyj New-AzManagementGroupDeployment cmdlet.

## PRZYKŁADY

### Przykład 1
```
PS C:\>Stop-AzManagementGroupDeployment -ManagementGroupId "myMG" -Name "deployment01"
```

To polecenie anuluje uruchomione wdrożenie "deployment01" w grupie zarządzania "myMG".

### Przykład 2
```
PS C:\>Get-AzManagementGroupDeployment -ManagementGroupId "myMG" -Name "deployment01" | Stop-AzManagementGroupDeployment
```

To polecenie pobiera wdrożenie "deployment01" w grupie zarządzania "myMG" i anuluje je.

# Update-AzPostgreSqlConfiguration

## SYNOPSIS
Aktualizuje konfigurację serwera.
Użyj Update-AzPostgreSqlServer, jeśli chcesz zaktualizować hasło AdministratorLoginPassword, sku itp.

## SKŁADNIA

### UpdateExpanded (Domyślna)
```
Update-AzPostgreSqlConfiguration -Name <String> -ResourceGroupName <String> -ServerName <String>
 [-SubscriptionId <String>] [-Source <String>] [-Value <String>] [-DefaultProfile <PSObject>] [-AsJob]
 [-NoWait] [-Confirm] [-WhatIf] [<CommonParameters>]
```

### UpdateViaIdentityExpanded
```
Update-AzPostgreSqlConfiguration -InputObject <IPostgreSqlIdentity> [-Source <String>] [-Value <String>]
 [-DefaultProfile <PSObject>] [-AsJob] [-NoWait] [-Confirm] [-WhatIf] [<CommonParameters>]
```

## OPIS
Aktualizuje konfigurację serwera.
Użyj Update-AzPostgreSqlServer, jeśli chcesz zaktualizować hasło AdministratorLoginPassword, sku itp.

## PRZYKŁADY

### Przykład 1. Aktualizowanie konfiguracji postgresql według nazwy
```powershell
PS C:\> Update-AzPostgreSqlConfiguration -Name intervalstyle -ResourceGroupName PostgreSqlTestRG -ServerName PostgreSqlTestServer -Value SQL_STANDARD

Name          Value
----          -----
intervalstyle SQL_STANDARD
```

To polecenie cmdlet aktualizuje konfigurację PostgreSql według nazwy.

### Przykład 2. Aktualizowanie konfiguracji postgresql przez tożsamość.
```powershell
PS C:\> $ID = "/subscriptions/<SubscriptionId>/resourceGroups/PostgreSqlTestRG/providers/Microsoft.DBforPostgreSQL/servers/PostgreSqlTestServer/configurations/deadlock_timeout"
PS C:\> Update-AzPostgreSqlConfiguration -InputObject $ID -Value 2000

Name             Value
----             -----
deadlock_timeout 2000
```

Te polecenia cmdlet aktualizują konfigurację PostgreSql według tożsamości.

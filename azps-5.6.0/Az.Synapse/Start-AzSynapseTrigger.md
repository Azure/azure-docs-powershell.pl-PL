# Start-AzSynapseTrigger

## SYNOPSIS
Uruchamia wyzwalacz w obszarze roboczym.

## SKŁADNIA

### StartByName (Domyślna)
```
Start-AzSynapseTrigger -WorkspaceName <String> -Name <String> [-PassThru] [-AsJob]
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

### StartByObject
```
Start-AzSynapseTrigger -WorkspaceObject <PSSynapseWorkspace> -Name <String> [-PassThru] [-AsJob]
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

### StartByInputObject
```
Start-AzSynapseTrigger -InputObject <PSTriggerResource> [-PassThru] [-AsJob]
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

## OPIS
Polecenie **cmdlet Start-AzSynapseTrigger** uruchamia wyzwalacz w obszarze roboczym. Jeśli wyzwalacz znajduje się w stanie "Zatrzymano", polecenie cmdlet uruchamia wyzwalacz i ostatecznie wywołuje potoki na podstawie jego definicji. Jeśli wyzwalacz znajduje się już w stanie "Uruchomiliśmy", to polecenie cmdlet nie ma wpływu.

## PRZYKŁADY

### Przykład 1
```powershell
PS C:\> Start-AzSynapseTrigger -WorkspaceName ContosoWorkspace -Name ContosoTrigger
```

Uruchamia wyzwalacz o nazwie ContosoTrigger w obszarze roboczym ContosoWorkspace.

### Przykład 2
```powershell
PS C:\> $ws = Get-AzSynapseWorkspace -Name ContosoWorkspace
PS C:\> $ws | Start-AzSynapseTrigger -Name ContosoTrigger
```

Uruchamia wyzwalacz o nazwie ContosoTrigger w obszarze roboczym ContosoWorkspace przez potok.

### Przykład 3
```powershell
PS C:\> $trigger = Get-AzSynapseTrigger -WorkspaceName ContosoWorkspace -Name ContosoTrigger
PS C:\> $trigger | Start-AzSynapseTrigger
```

Uruchamia wyzwalacz o nazwie ContosoTrigger w obszarze roboczym ContosoWorkspace przez potok.

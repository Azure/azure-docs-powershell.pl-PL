# Rename-AzContext

## SYNOPSIS
Zmień nazwę kontekstu platformy Azure.  Domyślnie konteksty są nazwane według konta użytkownika i subskrypcji.

## SKŁADNIA

### RenameByInputObject (Domyślne)
```
Rename-AzContext -InputObject <PSAzureContext> [-Force] [-PassThru] [-Scope <ContextModificationScope>]
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [-TargetName] <String> [<CommonParameters>]
```

### RenameByName
```
Rename-AzContext [-Force] [-PassThru] [-Scope <ContextModificationScope>]
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [-SourceName] <String> [-TargetName] <String>
 [<CommonParameters>]
```

## OPIS
Zmień nazwę kontekstu platformy Azure.  Domyślnie konteksty są nazwane według konta użytkownika i subskrypcji.

## PRZYKŁADY

### Przykład 1. Zmienianie nazwy kontekstu przy użyciu nazwanych parametrów
```powershell
PS C:\> Rename-AzContext -SourceName "[user1@contoso.org; 12345-6789-2345-3567890]" -TargetName "Work"
```

Zmień nazwę kontekstu dla ' na subskrypcję user1@contoso.org '12345-6789-2345-3567890' na "Służbowy".  Po użyciu tego polecenia będzie można kierować kontekst za pomocą polecenia "Select-AzContext Work".  Wartości parametru "SourceName" można przechodzić za pomocą klawisza Tab, używając zakończenia karty.

### Przykład 2. Zmienianie nazwy kontekstu przy użyciu parametrów pozytowych
```powershell
PS C:\> Rename-AzContext "My context" "Work"
```

Zmień nazwę kontekstu o nazwie "Mój kontekst" na "Praca".  Po użyciu tego polecenia będzie można kierować kontekst przy użyciu polecenia Select-AzContext Work

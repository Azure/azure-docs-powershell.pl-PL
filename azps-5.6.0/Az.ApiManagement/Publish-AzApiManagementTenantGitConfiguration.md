# Publish-AzApiManagementTenantGitConfiguration

## SYNOPSIS
Publikuje zmiany z gałęzi Git do bazy danych konfiguracji.

## SKŁADNIA

```
Publish-AzApiManagementTenantGitConfiguration -Context <PsApiManagementContext> -Branch <String> [-Force]
 [-ValidateOnly] [-PassThru] [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm]
 [<CommonParameters>]
```

## OPIS
Polecenie cmdlet **Publish-AzApiManagementTenantGitConfiguration** publikuje zmiany z gałęzi Git do bazy danych konfiguracji.
Możesz również sprawdzić poprawność zmian w rozgałęzieniach Git bez publikowania.

## PRZYKŁADY

### Przykład 1. Wdrażanie zmian Git
```
PS C:\>$apimContext = New-AzApiManagementContext -ResourceGroupName "Api-Default-WestUS" -ServiceName "contoso"
PS C:\>Publish-AzApiManagementTenantGitConfiguration -Context $apimContext -Branch 'master' -PassThru
```

To polecenie publikuje zmiany z określonej gałęzi w bazie danych konfiguracji.

### Przykład 2. Sprawdzanie poprawności zmian Git
```
PS C:\>$apimContext = New-AzApiManagementContext -ResourceGroupName "Api-Default-WestUS" -ServiceName "contoso"
PS C:\>Publish-AzApiManagementTenantGitConfiguration -Context $apimContext -Branch 'master' -ValidateOnly -PassThru
```

To polecenie sprawdza poprawność zmian w rozgałęzieniu Git w bazie danych konfiguracji.
Nie powoduje publikowania zmian.

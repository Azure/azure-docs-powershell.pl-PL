# Move-AzExpressRouteCircuit

## SYNOPSIS
Przenosi obwód usługi ExpressRoute z klasycznego modelu wdrażania do modelu wdrażania Menedżera zasobów.

## SKŁADNIA

```
Move-AzExpressRouteCircuit -Name <String> -ResourceGroupName <String> -Location <String> -ServiceKey <String>
 [-Tag <Hashtable>] [-Force] [-AsJob] [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm]
 [<CommonParameters>]
```

## OPIS
Polecenie **cmdlet Move-AzExpressRouteCircuit** przenosi obwód usługi ExpressRoute z klasycznego modelu wdrażania do modelu wdrażania menedżera zasobów. Po zakończeniu przenoszenia obwód usługi ExpressRoute zachowuje się i wykonuje tak samo jak każdy inny obwód usługi ExpressRoute utworzony w modelu wdrażania menedżera zasobów. W ramach tej operacji nie są przenoszone linki obwodów, sieci wirtualne ani bramy VPN. Te zasoby należy ponownie skonfigurować po zakończeniu przenoszenia.

## PRZYKŁADY

### Przykład 1. Przenoszenie obwodu usługi ExpressRoute do modelu wdrażania Menedżera zasobów
```
Move-AzExpressRouteCircuit -Name $CircuitName -ResourceGroupName $RG -Location $Location -ServiceKey $ServiceKey
```

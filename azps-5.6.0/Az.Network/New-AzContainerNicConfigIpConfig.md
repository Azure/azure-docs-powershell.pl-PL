# New-AzContainerNicConfigIpConfig

## SYNOPSIS
Tworzy obiekt konfiguracji adresu IP konfiguracji kontenera.

## SKŁADNIA

```
New-AzContainerNicConfigIpConfig -Name <String> -Subnet <PSSubnet> [-SubnetId <String>]
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

## OPIS
Polecenie **cmdlet New-AzContainerNicConfigIpConfig** tworzy nową konfigurację ip interfejsu sieciowego kontenera. 

## PRZYKŁADY

### Przykład 1
```powershell
$subnet = New-AzVirtualNetworkSubnetConfig -Name subnet -AddressPrefix 10.0.1.0/24

$vnet = New-AzVirtualNetwork -Name vnet -ResourceGroupName rg1 -Location "West US" -AddressPrefix 10.0.0.0/16 -Subnet $subnet

$containerNicConfigIpConfig = New-AzContainerNicConfigIpConfig -Name ipconfigprofile1 -Subnet $vnet.Subnets[0]

$containerNicConfig = New-AzContainerNicConfig -Name cnic -IpConfiguration containerNicConfigIpConfig

$networkProfile = New-AzNetworkProfile -Name np1 -Location "West US" -ResourceGroupName rg1 -ContainerNetworkInterfaceConfiguration $containerNicConfig
```

Pierwsze dwa polecenia tworzą i inicjują podsieci vnet oraz podsieci. Trzecie polecenie tworzy profil konfiguracji nic ip container nic odwołujący się do utworzonej podsieci. Czwarte polecenie tworzy konfigurację kontenera interfejsu sieciowego, która dostarcza profil konfiguracji adresu IP utworzony w poprzednim poleceniu. Na koniec piąte polecenie tworzy profil sieciowy zainicjowany przy użyciu konfiguracji interfejsu sieciowego kontenerów przechowywanej w $containerNicConfig.

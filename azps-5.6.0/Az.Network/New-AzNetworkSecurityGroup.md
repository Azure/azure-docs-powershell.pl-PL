# New-AzNetworkSecurityGroup

## SYNOPSIS
Tworzy grupę zabezpieczeń sieciowych.

## SKŁADNIA

```
New-AzNetworkSecurityGroup -Name <String> -ResourceGroupName <String> -Location <String>
 [-SecurityRules <PSSecurityRule[]>] [-Tag <Hashtable>] [-Force] [-AsJob]
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

## OPIS
Polecenie **cmdlet New-AzNetworkSecurityGroup** tworzy grupę zabezpieczeń sieci platformy Azure.

## PRZYKŁADY

### Przykład 1. Tworzenie nowej grupy zabezpieczeń sieciowych
```powershell
New-AzNetworkSecurityGroup -Name "nsg1" -ResourceGroupName "rg1"  -Location  "westus"
```

To polecenie tworzy nową grupę zabezpieczeń sieci platformy Azure o nazwie "nsg1" w grupie zasobów "rg1" w lokalizacji "westus".

### Przykład 2. Tworzenie szczegółowej grupy zabezpieczeń sieci
```powershell
$rule1 = New-AzNetworkSecurityRuleConfig -Name rdp-rule -Description "Allow RDP" `
    -Access Allow -Protocol Tcp -Direction Inbound -Priority 100 -SourceAddressPrefix `
    Internet -SourcePortRange * -DestinationAddressPrefix * -DestinationPortRange 3389

$rule2 = New-AzNetworkSecurityRuleConfig -Name web-rule -Description "Allow HTTP" `
    -Access Allow -Protocol Tcp -Direction Inbound -Priority 101 -SourceAddressPrefix `
    Internet -SourcePortRange * -DestinationAddressPrefix * -DestinationPortRange 80

$nsg = New-AzNetworkSecurityGroup -ResourceGroupName TestRG -Location westus -Name `
    "NSG-FrontEnd" -SecurityRules $rule1,$rule2
```

Krok:1 Utwórz regułę zabezpieczeń zezwalającą na dostęp z Internetu do portu 3389.
Krok:2 Tworzenie reguły zabezpieczeń umożliwiającej dostęp z Internetu do portu 80.
Krok:3 Dodawanie reguł utworzonych powyżej do nowej nazwy NSG o nazwie NSG-FrontEnd.

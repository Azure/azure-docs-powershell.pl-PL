# Initialize-AzRecoveryServicesBackupProtectableItem

## SYNOPSIS
To polecenie wyzwala odnajdowanie elementów niechronionych danego typu obciążenia pracą w danym kontenerze. Jeśli aplikacja bazy danych nie jest automatycznie chroniona, użyj tego polecenia do odnajdowania nowych znaków DB po każdym dodaniu i kontynuowaniu ich ochrony.

## SKŁADNIA

```
Initialize-AzRecoveryServicesBackupProtectableItem [-Container] <ContainerBase> [-WorkloadType] <WorkloadType>
 [-PassThru] [-VaultId <String>] [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm]
 [<CommonParameters>]
```

## OPIS
zapytania cmdlet dotyczące określonych obciążeń w kontenerze. Wyzwala to operację, która tworzy elementy, które można chronić.

## PRZYKŁADY

### Przykład 1
```powershell
PS C:\> Initialize-AzRecoveryServicesProtectableItem -Container $Container -WorkloadType "MSSQL"
```

Polecenie cmdlet wykonuje operację odnajdowania nowych elementów, które można chronić.

### Przykład 2

To polecenie wyzwala odnajdowanie elementów niechronionych danego typu obciążenia pracą w danym kontenerze. (wygenerowane automatycznie)

```powershell <!-- Aladdin Generated Example --> 
Initialize-AzRecoveryServicesBackupProtectableItem -Container $Cont[0] -VaultId $vault.ID -WorkloadType AzureVM
```

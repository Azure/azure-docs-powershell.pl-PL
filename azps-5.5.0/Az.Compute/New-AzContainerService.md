# New-AzContainerService

## SYNOPSIS
Tworzy usługę kontenerową.

## SKŁADNIA

```
New-AzContainerService [-ResourceGroupName] <String> [-Name] <String> [-ContainerService] <PSContainerService>
 [-AsJob] [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

## OPIS
Polecenie **cmdlet New-AzContainerService** tworzy usługę kontenerową.
Określ obiekt usługi kontenera, który można utworzyć za pomocą New-AzContainerServiceConfig cmdlet.

## PRZYKŁADY

### Przykład 1. Tworzenie usługi kontenerowej
```
PS C:\> New-AzResourceGroup -Name "ResourceGroup17" -Location "East US" -Force
PS C:\> $Container = New-AzContainerServiceConfig -Location "East US" -OrchestratorType "DCOS" -MasterDnsPrefix "MasterResourceGroup17" -AdminUsername "acslinuxadmin" -SshPublicKey "<ssh-key>" | Add-AzContainerServiceAgentPoolProfile -Name "AgentPool01" -VmSize "Standard_A1" -DnsPrefix "APResourceGroup17" -Count 2
PS C:\> New-AzContainerService -ResourceGroupName "ResourceGroup17" -Name "CSResourceGroup17" -ContainerService $Container
```

Pierwsze polecenie tworzy grupę zasobów o nazwie Grupa Zasobów17 w określonej lokalizacji.
Aby uzyskać więcej informacji, zobacz New-AzResourceGroup cmdlet.
Drugie polecenie tworzy kontener, a następnie przechowuje go w $Container danych.
Aby uzyskać więcej informacji, zobacz New-AzContainerServiceConfig cmdlet.
Końcowe polecenie tworzy usługę kontenera dla kontenera przechowywanego w $Container.
Usługa nosi nazwę csResourceGroup17.

# New-AzLogicApp

## SYNOPSIS
Tworzy aplikację logiczną w grupie zasobów.

## SKŁADNIA

### LogicAppWithDefinitionParameterSet
```
New-AzLogicApp -ResourceGroupName <String> -Name <String> -Location <String> [-State <String>]
 -Definition <Object> [-IntegrationAccountId <String>] [-Parameters <Object>] [-ParameterFilePath <String>]
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

### LogicAppWithDefinitionFileParameterSet
```
New-AzLogicApp -ResourceGroupName <String> -Name <String> -Location <String> [-State <String>]
 -DefinitionFilePath <String> [-IntegrationAccountId <String>] [-Parameters <Object>]
 [-ParameterFilePath <String>] [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm]
 [<CommonParameters>]
```

## OPIS
Polecenie **cmdlet New-AzLogicApp** tworzy aplikację logiki przy użyciu funkcji aplikacje logic.
Aplikacja logiczna to zbiór akcji lub wyzwalaczy zdefiniowanych w definicji aplikacji logiki.
To polecenie cmdlet zwraca obiekt **przepływu** pracy.
Możesz utworzyć aplikację logiki, określając nazwę, lokalizację, definicję aplikacji logiki, grupę zasobów i plan.
Definicja i parametry aplikacji logiki są sformatowane w notacji obiektów JavaScript (JSON).
Jako szablonu definicji i parametrów można użyć aplikacji logiki.
Ten moduł obsługuje parametry dynamiczne.
Aby użyć parametru dynamicznego, wpisz go w poleceniu.
Aby wykryć nazwy parametrów dynamicznych, wpisz łącznik (-) po nazwie polecenia cmdlet, a następnie naciskaj klawisz Tab, aby przechodzić przez dostępne parametry.
Jeśli pominiesz wymagany parametr szablonu, polecenie cmdlet wyświetli monit o wartość.
Wartości plików parametrów szablonu określone w wierszu polecenia mają pierwszeństwo przed wartościami parametrów szablonu w obiekcie parametru szablonu.

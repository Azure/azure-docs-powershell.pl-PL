# Invoke-AzIotHubManualFailover

## SYNOPSIS
Wywołaj proces trybu failover dla centrum IoT do regionu odzyskiwania po awarii sparowanego geograficznie.

## SKŁADNIA

### ResourceSet (Default)
```
Invoke-AzIotHubManualFailover [-ResourceGroupName] <String> [-Name] <String> [-AsJob] [-PassThru]
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

### InputObjectSet
```
Invoke-AzIotHubManualFailover [-InputObject] <PSIotHub> [-AsJob] [-PassThru]
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

### ResourceIdSet
```
Invoke-AzIotHubManualFailover [-ResourceId] <String> [-AsJob] [-PassThru]
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

## OPIS
Spowoduje to wyzwolenie trybu failover Twojego centrum IoT do lokalizacji pomocniczej. Ta akcja spowoduje spowolnienie czasu i utraty telemetrii rozwiązania. Jest to długo uruchomiona operacja, która może potrwać kilka minut. Podczas korzystania z urządzenia należy zachować ostrożność.

## PRZYKŁADY

### Przykład 1
```powershell
PS C:\> Invoke-AzIotHubManualFailover -ResourceGroupName "myresourcegroup" -Name "myiothub"
```

Inicjowanie procesu trybu failover centrum IoT "myiothub".

### Przykład 2
```powershell
PS C:\> Invoke-AzIotHubManualFailover -ResourceGroupName "myresourcegroup" -Name "myiothub" -AsJob
```

Inicjowanie procesu failover centrum IoT Hub "myiothub" w tle.

# Invoke-AzStorageSyncFileRecall

## SYNOPSIS
To polecenie przypomina wszystkie pliki warstwowe z powrotem na dysk lokalny.

## SKŁADNIA

### StringParameterSet (Domyślne)
```
Invoke-AzStorageSyncFileRecall [-ResourceGroupName] <String> [-StorageSyncServiceName] <String>
 [-SyncGroupName] <String> [-Name] <String> [-Pattern <String>] [-RecallPath <String>] [-PassThru] [-AsJob]
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

### ResourceIdParameterSet
```
Invoke-AzStorageSyncFileRecall [-ResourceId] <String> [-Pattern <String>] [-RecallPath <String>] [-PassThru]
 [-AsJob] [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

### ObjectParameterSet
```
Invoke-AzStorageSyncFileRecall [-InputObject] <PSServerEndpoint> [-Pattern <String>] [-RecallPath <String>]
 [-PassThru] [-AsJob] [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

## OPIS
Gdy dla punktu końcowego serwera (określonej lokalizacji na zarejestrowanym serwerze) jest włączone zarządzanie warstwami w chmurze, to polecenie może zostać użyte do odwoływania wszystkich plików warstwowych na dysk lokalny. Przed rozpoczęciem odwoływania zdecydowanie zalecane jest wyłączenie warstw w chmurze dla tego punktu końcowego serwera. Jeśli wciąż trwa warstwowanie, odwoływanie może prowadzić do tego, że inne pliki są warstwowe, co może okazać się chybne, aby osiągnąć cel, jaki ma zapewnić cała zawartość, która znajduje się na dysku lokalnym.

## PRZYKŁADY

### Przykład 1
```powershell
PS C:\> Invoke-AzStorageSyncFileRecall -ResourceGroupName "myResourceGroup" -StorageSyncServiceName "myStorageSyncServiceName" -SyncGroupName "mySyncGroupName" -ServerEndpointName "myServerEndpointName"
```

To polecenie cyklicznie przypomina wszystkie pliki warstwowe znajdujące się w ścieżce głównej określonego punktu końcowego serwera.

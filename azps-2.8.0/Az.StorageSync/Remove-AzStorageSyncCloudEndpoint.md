# Remove-AzStorageSyncCloudEndpoint

## STRESZCZENIe
To polecenie spowoduje usunięcie określonego punktu końcowego chmury z grupy synchronizacji. Bez co najmniej jednego punktu końcowego w chmurze nie można zsynchronizować innych punktów końcowych serwera w tej grupie synchronizacji.

## POLECENIA

### StringParameterSet (domyślny)
```
Remove-AzStorageSyncCloudEndpoint [-ResourceGroupName] <String> [-StorageSyncServiceName] <String>
 [-SyncGroupName] <String> [-Name] <String> [-Force] [-PassThru] [-AsJob]
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

### InputObjectParameterSet
```
Remove-AzStorageSyncCloudEndpoint [-InputObject] <PSCloudEndpoint> [-Force] [-PassThru] [-AsJob]
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

### ResourceIdParameterSet
```
Remove-AzStorageSyncCloudEndpoint [-ResourceId] <String> [-Force] [-PassThru] [-AsJob]
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

## Opis
To polecenie spowoduje usunięcie określonego punktu końcowego chmury z grupy synchronizacji. Usługa Azure File Share odwołania do punktów końcowych w chmurze pozostaną nienaruszone przez ten proces. To polecenie jest przeznaczone tylko do likwidacji. Usunięcie punktu końcowego w chmurze jest operacją niszczącą. Punkty końcowe serwera nie mogą być synchronizowane bez co najmniej jednego punktu końcowego w chmurze. Nie należy wykonywać tej operacji, aby rozwiązać problemy z synchronizacją. Jeśli ten udział plików w usłudze Azure zostanie dodany ponownie do tej samej grupy synchronizacji, w ramach punktu końcowego w chmurze, może on powodować konflikty z plikami i inne niezamierzone konsekwencje.

## Przykłady

### Przykład 1
```powershell
PS C:\> Remove-AzStorageSyncCloudEndpoint -Force -ResourceGroupName "myResourceGroup" -StorageSyncServiceName "myStorageSyncServiceName" -SyncGroupName "mySyncGroupName" -Name "myCloudEndpointName"
```

To polecenie spowoduje usunięcie punktu końcowego w chmurze.

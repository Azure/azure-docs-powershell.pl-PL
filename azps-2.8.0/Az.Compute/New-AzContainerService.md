# New-AzContainerService

## STRESZCZENIe
Tworzy usługę kontenera.

## POLECENIA

```
New-AzContainerService [-ResourceGroupName] <String> [-Name] <String> [-ContainerService] <PSContainerService>
 [-AsJob] [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

## Opis
Polecenie cmdlet **New-AzContainerService** tworzy usługę kontenera.
Określ obiekt usługi kontenera, który możesz utworzyć przy użyciu New-AzContainerServiceConfig polecenia cmdlet.

## Przykłady

### Przykład 1: Tworzenie usługi kontenera
```
PS C:\> New-AzResourceGroup -Name "ResourceGroup17" -Location "East US" -Force
PS C:\> $Container = New-AzContainerServiceConfig -Location "East US" -OrchestratorType "DCOS" -MasterDnsPrefix "MasterResourceGroup17" -AdminUsername "acslinuxadmin" -SshPublicKey "<ssh-key>" | Add-AzContainerServiceAgentPoolProfile -Name "AgentPool01" -VmSize "Standard_A1" -DnsPrefix "APResourceGroup17" -Count 2
PS C:\> New-AzContainerService -ResourceGroupName "ResourceGroup17" -Name "CSResourceGroup17" -ContainerService $Container
```

Pierwsze polecenie tworzy grupę zasobów o nazwie ResourceGroup17 w określonej lokalizacji.
Aby uzyskać więcej informacji, zobacz polecenie cmdlet New-AzResourceGroup.
Drugie polecenie tworzy kontener, a następnie zapisuje go w zmiennej $Container.
Aby uzyskać więcej informacji, zobacz polecenie cmdlet New-AzContainerServiceConfig.
Polecenie ostatnie tworzy usługę kontenera dla kontenera przechowywanego w $Container.
Usługa ma nazwę csResourceGroup17.

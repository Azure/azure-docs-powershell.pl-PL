# Add-AzSqlInstanceKeyVaultKey

## STRESZCZENIe
Dodaje klucz magazynu kluczy do udostępnionego wystąpienia zarządzanego. 

## POLECENIA

### AddAzureRmSqlInstanceKeyVaultKeyDefaultParameterSet (domyślny)
```
Add-AzSqlInstanceKeyVaultKey [-ResourceGroupName] <String> [-InstanceName] <String> [-KeyId] <String>
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

### AddAzureRmSqlInstanceKeyVaultKeyInputObjectParameterSet
```
Add-AzSqlInstanceKeyVaultKey [-Instance] <AzureSqlManagedInstanceModel> [-KeyId] <String>
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

### AddAzureRmSqlInstanceKeyVaultKeyResourceIdParameterSet
```
Add-AzSqlInstanceKeyVaultKey [-InstanceResourceId] <String> [-KeyId] <String>
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

## Opis
Polecenie cmdlet Add-AzSqlInstanceKeyVaultKey dodaje klucz magazynu kluczy do udostępnionego wystąpienia zarządzanego. W zarządzanym wystąpieniu muszą być przyznane uprawnienia Get, wrapKey, unwrapKey ', aby udzielić uprawnień do wystąpienia zarządzanego, należy użyć poniższego skryptu.
$managedInstance = Get-AzSqlInstance-Name "ContosoManagedInstanceName"-ResourceGroupName "ContosoResourceGroup" Set-AzKeyVaultAccessPolicy--ID-ObjectId $managedInstance. Identity. PrincipalId-PermissionsToKeys Get, wrapKey, unwrapKey

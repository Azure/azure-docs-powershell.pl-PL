# Remove-AzBatchPool

## STRESZCZENIe
Usuwa określoną pulę partii.

## POLECENIA

```
Remove-AzBatchPool [-Id] <String> [-Force] -BatchContext <BatchAccountContext>
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

## Opis
Polecenie cmdlet **Remove-AzBatchPool** usuwa określoną pulę usługi Azure Batch.
Zostanie wyświetlony monit o potwierdzenie, chyba że zostanie użyty parametr *Force* .

## Przykłady

### Przykład 1: Usuwanie puli partii według identyfikatora puli
```
PS C:\>Remove-AzBatchPool -Id "MyPool" -BatchContext $Context
```

To polecenie usuwa pulę o IDENTYFIKATORze moja Pula.
Użytkownik jest monitowany o potwierdzenie przed wykonaniem operacji usuwania.

### Przykład 2: Usuwanie wszystkich pul partii według siły
```
PS C:\>Get-AzBatchPool -BatchContext $Context | Remove-AzBatchPool -Force -BatchContext $Context
```

To polecenie usuwa wszystkie pule partii.
Ponieważ parametr *Force* jest obecny, monit o potwierdzenie jest pomijany.

## PARAMETRÓW

### -BatchContext
Określa wystąpienie **BatchAccountContext** , którego używa polecenie cmdlet w celu interakcji z usługą przetwarzania wsadowego.
Jeśli używasz polecenia cmdlet Get-AzBatchAccount, aby uzyskać BatchAccountContext, uwierzytelnianie usługi Azure Active Directory zostanie użyte podczas interakcji z usługą przetwarzania wsadowego. Aby zamiast tego użyć uwierzytelniania opartego na kluczu udostępnionym, użyj polecenia cmdlet Get-AzBatchAccountKey, aby uzyskać obiekt BatchAccountContext z wypełnionymi klawiszami dostępu. W przypadku korzystania z uwierzytelniania za pomocą klucza udostępnionego domyślnie jest używany podstawowy klucz dostępu. Aby zmienić klucz, który ma być używany, ustaw właściwość BatchAccountContext. KeyInUse.

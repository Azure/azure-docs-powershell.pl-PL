# New-AzureAutomationVariable

## STRESZCZENIe

Tworzy zmienną automatyzacji.

## POLECENIA

```
New-AzureAutomationVariable -Name <String> -Encrypted <Boolean> [-Description <String>] [-Value <Object>]
 -AutomationAccountName <String> [-Profile <AzureSMProfile>] [<CommonParameters>]
```

## Opis

[!INCLUDE [aa-deprecation](../include/aa-deprecation.md)]

Polecenie cmdlet **New-AzureAutomationVariable** tworzy zmienną w automatyzacji Microsoft Azure.

## Przykłady

### Przykład 1. Tworzenie nowej zmiennej z wartością prostą
```
PS C:\> New-AzureAutomationVariable -AutomationAccountName "Contoso17" -Name "MyStringVariable" -Encrypted $False -Value "My String"
```

To polecenie tworzy nową zmienną o nazwie MyStringVariable z wartością ciągu na koncie usługi Azure Automation o nazwie Contoso17.

### Przykład 2: Tworzenie nowej zmiennej o wartości złożonej
```
PS C:\> $vm = Get-AzureVM -ServiceName "MyVM" -Name "MyVM"
PS C:\> New-AzureAutomationVariable -AutomationAccountName "Contoso17" -Name "MyComplexVariable" -Encrypted $False -Value $vm
```

Te polecenia tworzą nową zmienną o nazwie MyComplexVariable na koncie automatyzacji o nazwie Contoso17.
W tym przypadku jest wykorzystywany obiekt złożony, a w tym przypadku obiekt maszyny wirtualnej.

# New-WAPackStaticIPAddressPool

## STRESZCZENIe
Tworzy statyczną pulę adresów IP.

## POLECENIA

```
New-WAPackStaticIPAddressPool -VMSubnet <VMSubnet> -Name <String> -IPAddressRangeStart <String>
 -IPAddressRangeEnd <String> [-Profile <AzureSMProfile>] [<CommonParameters>]
```

## Opis
Te tematy są przestarzałe i zostaną usunięte w przyszłości.
W tym temacie opisano polecenie cmdlet w wersji 0.8.1 modułu PowerShell platformy Microsoft Azure.
Aby sprawdzić używaną wersję modułu, należy wpisać ją w konsoli programu Azure PowerShell `(Get-Module -Name Azure).Version` .

Polecenie cmdlet **New-WAPackStaticIPAddressPool** tworzy statyczną pulę adresów IP.

## Przykłady

### Przykład 1. Tworzenie puli statycznych adresów IP
```
PS C:\> $VNet = Get-WAPackVNet -Name "ContosoVNet01"
PS C:\> $VMSubnet = Get-WAPackVMSubnet -VNet $VNet -Name "ContosoVMSubnet01"
PS C:\> New-WAPackStaticIpAddressPool ?VMSubnet $VMSubnet -Name "ContosoStaticIpAddressPool01" -IPAddressRangeStart "192.168.1.0" -IPAddressRangeEnd "192.168.1.10"
```

Pierwsze polecenie najpierw pobiera sieć maszyny wirtualnej, do której chcemy dodać pulę statycznych adresów IP.
Ta sieć maszyny wirtualnej nosi nazwę ContosoVNet01.

W drugim poleceniu jest używana uprzednio pobrana sieć maszyny wirtualnej w celu pobrania podsieci maszyny wirtualnej o nazwie ContosoVMSubnet01, do której chcemy dodać pulę statycznych adresów IP.

Ostatnie polecenie tworzy nową statyczną pulę adresów IP z nazwą ContosoStaticIpAddressPool01 oraz początkiem zakresu od 192.168.1.0 i 192.168.1.10 końca zakresu.

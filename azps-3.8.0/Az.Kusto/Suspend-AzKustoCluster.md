# Suspend-AzKustoCluster

## STRESZCZENIe
Zawieszanie istniejącego klastra Kusto.

## POLECENIA

### ByNameAndResourceGroup (domyślny)
```
Suspend-AzKustoCluster [-ResourceGroupName] <String> [-Name] <String> [-PassThru]
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

### ByResourceId
```
Suspend-AzKustoCluster [-ResourceId] <String> [-PassThru] [-DefaultProfile <IAzureContextContainer>] [-WhatIf]
 [-Confirm] [<CommonParameters>]
```

### ByInputObject
```
Suspend-AzKustoCluster [-InputObject] <PSKustoCluster> [-PassThru] [-DefaultProfile <IAzureContextContainer>]
 [-WhatIf] [-Confirm] [<CommonParameters>]
```

## Opis
Zawieszanie istniejącego klastra Kusto.

## Przykłady

### Przykład 1-wstrzymywanie istniejącego klastra Kusto według nazwy

```
PS C:\> Suspend-AzKustoCluster -ResourceGroupName testrg -Name mykustocluster
```

Powyższe polecenie zawiesza klaster Kusto o nazwie "mykustocluster" znaleziony w grupie zasobów "testrg".

### Przykład 2 — zawieszanie istniejącego klastra Kusto za pomocą połączeń rurowych

```
PS C:\> Get-AzKustoCluster -ResourceGroupName testrg -Name mykustocluster | Suspend-AzKustoCluster
```

Powyższe polecenie pobiera klaster Kusto o nazwie "mykustocluster" znajdujący się w grupie zasobów "testrg" przy użyciu `Get-AzKustoCluster` polecenia cmdlet, a następnie przekazuje wynik, aby `Suspend-AzKustoCluster` go wstrzymać.
